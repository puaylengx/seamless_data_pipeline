1. Prerequisites

ตรวจสอบให้ครบก่อนเริ่ม:

Docker Desktop

ติดตั้ง Docker Desktop (Windows / macOS) หรือ Docker Engine (Linux)
Memory ≥ 4 GB, CPU ≥ 2 cores, Disk ≥ 10 GB ว่าง

docker --version
docker compose version

Git

git --version

Access ถึง Database ต้นทาง

ต้องเชื่อมต่อ MSSQL (DGSI) และ PostgreSQL (PaperCut) ได้จาก host หรือ container

Google Cloud Service Account

ต้องการสำหรับ BigQuery pipelines: student_information_pipeline และ printer_solution_pipeline
วาง JSON key ไว้ที่ config/secrets/

2. Clone / เตรียมโปรเจกต์

git clone <repo-url> seamless_data_pipeline
cd seamless_data_pipeline

โครงสร้างโฟลเดอร์หลักที่ต้องมี:

seamless_data_pipeline/
├── dags/               ← DAG files + source modules
│   └── src/
│       ├── config.py
│       ├── dgsi/       ← finance_invoice, staging_student
│       ├── education/  ← student_information
│       ├── information_tech/ ← printer_solution
│       └── helpers/    ← audit, emailer, logger
├── config/
│   ├── airflow.cfg
│   └── secrets/        ← GCP service account keys (ห้าม commit)
├── plugins/
├── .env                ← Environment variables
├── docker-compose.yaml
└── requirements.txt

3. ตั้งค่า .env

สร้าง / แก้ไขไฟล์ .env ที่ root ของโปรเจกต์:

# UID ของ Airflow container (Linux ให้รัน: echo $UID)
AIRFLOW_UID=50000

# ===================== SMTP (Email Notification) =====================
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_STARTTLS=false
SMTP_SSL=true
SMTP_USER=your.email@gmail.com
SMTP_PASSWORD=your_app_password      # Gmail App Password (16 หลัก)
SMTP_MAIL_FROM=Airflow Notification
ALERT_EMAILS=recipient@example.com

# ===================== Finance Invoice Job Config ====================
FIN_INV_TARGET_SCHEMA=dbo
FIN_INV_TARGET_TABLE=finance_invoice_20251110
FIN_INV_KEY_COL=invoiceId
FIN_INV_BATCH_SIZE=1000
⚠️
.env มี credentials ห้าม commit ขึ้น Git!
ตรวจสอบว่า .gitignore มี .env และ config/secrets/ อยู่แล้ว

6. รัน Docker Compose

Initialize Airflow (ครั้งแรกเท่านั้น)

docker compose up airflow-init

รอจนเห็น exited with code 0 — ระบบจะ migrate DB และสร้าง admin user

รัน Services ทั้งหมด

docker compose up -d

รอประมาณ 1-2 นาทีจนทุก service healthy

ตรวจสอบ Status

docker compose ps

ทุก service ควรแสดง healthy หรือ running

เข้า Airflow Web UI

เปิด browser ไปที่ http://localhost:8080
username: airflow   password: airflow

(Optional) Flower — Monitor Celery Workers

docker compose --profile flower up -d

เข้าดูที่ http://localhost:5555

⚠️
Windows — AIRFLOW_UID
บน Windows ไม่ต้องเปลี่ยน AIRFLOW_UID จากค่า default 50000 ใน .env

7. ตรวจสอบ Services

# ดู log ของ scheduler
docker compose logs -f airflow-scheduler

# ดู log ของ worker
docker compose logs -f airflow-worker

# ดู log เฉพาะ service เดียว
docker compose logs --tail=50 airflow-apiserver

8. หยุด / ลบ Container

# หยุด containers (เก็บ volumes ไว้)
docker compose down

# หยุด + ลบ volumes ทั้งหมด (ล้าง Airflow DB ด้วย)
docker compose down --volumes --remove-orphans
🚨
--volumes จะลบ PostgreSQL metadata ของ Airflow ทั้งหมด (DAG runs, logs, connections, variables) — ใช้เมื่อต้องการ reset เท่านั้น

Tips & Troubleshoot

เพิ่ม Python packages

packages หลักกำหนดใน requirements.txt แต่ใน Docker Compose ใช้ _PIP_ADDITIONAL_REQUIREMENTS ใน docker-compose.yaml:
สำหรับ production ควร build custom Docker image แทน

DAG ไม่โชว์ใน UI

# ตรวจสอบ DAG parse error
docker compose exec airflow-scheduler airflow dags list-import-errors

ดู logs ของ Task run

ไปที่ Airflow UI → DAG → Graph → คลิก Task → Log

รัน DAG ด้วย CLI

docker compose exec airflow-scheduler \
  airflow dags trigger student_information_pipeline

ปัญหา Memory ไม่พอ (Docker Desktop)

ไปที่ Docker Desktop → Settings → Resources → Memory: เพิ่มเป็น ≥ 4 GB

Gmail App Password

เปิด 2-Step Verification ก่อน → Google Account → Security → App passwords → สร้าง password 16 หลักสำหรับ Airflow
