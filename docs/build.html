Prerequisites

ตรวจสอบให้ครบก่อนเริ่ม setup ระบบต้องการ resource ขั้นต่ำดังนี้

Docker Desktop

ติดตั้ง Docker Desktop (Windows / macOS) หรือ Docker Engine (Linux)
Memory ≥ 4 GB  ·  CPU ≥ 2 cores  ·  Disk ≥ 10 GB ว่าง

docker --version
docker compose version

Git

git --version

Network Access ถึง Database ต้นทาง

ต้องเชื่อมต่อ MSSQL (SKY) และ PostgreSQL (PaperCut) ได้จาก host หรือ Docker container

Google Cloud Service Account

ต้องการสำหรับ BigQuery pipelines: student_information_pipeline และ printer_solution_pipeline
วาง JSON key ไว้ที่ config/secrets/

เลือกวิธีเริ่มต้น — มี 2 เส้นทาง:

🆕

Path A — เริ่มจากศูนย์

ยังไม่มีไฟล์ docker-compose.yaml หรือ Git repo
ดาวน์โหลดและตั้งค่าทุกอย่างด้วยตัวเอง

→ ไปที่ Section 2
📦

Path B — มี Repository อยู่แล้ว

มี Git repo ที่มีไฟล์ docker-compose.yaml และ DAGs ครบแล้ว
แค่ Clone แล้วตั้งค่า credentials

→ ไปที่ Section 3

ทั้งสองเส้นทางมาบรรจบกันที่ Section 4 (ตั้งค่า .env) เป็นต้นไป

2

เริ่มจากศูนย์ — ดาวน์โหลด docker-compose.yaml

Path A — ไม่มี Project มาก่อน

ทำตามขั้นตอนนี้เมื่อยังไม่มีไฟล์ใดๆ ในเครื่อง เริ่มตั้งแต่สร้างโฟลเดอร์ไปจนถึงได้ Airflow รันได้จริง

สร้างโฟลเดอร์โปรเจกต์

# Linux / macOS / Git Bash
mkdir seamless_data_pipeline
cd seamless_data_pipeline
# Windows PowerShell
New-Item -ItemType Directory seamless_data_pipeline
Set-Location seamless_data_pipeline

ดาวน์โหลด docker-compose.yaml จาก Apache Airflow

ไฟล์นี้คือ Official Airflow 3.1.8 CeleryExecutor template ที่ Apache ให้มา:

# Linux / macOS
curl -LfO 'https://airflow.apache.org/docs/apache-airflow/3.1.8/docker-compose.yaml'
# Windows PowerShell
Invoke-WebRequest `
  -Uri 'https://airflow.apache.org/docs/apache-airflow/3.1.8/docker-compose.yaml' `
  -OutFile docker-compose.yaml
ℹ️
ไฟล์ที่ได้มาจาก Apache มี CeleryExecutor + Redis + PostgreSQL พร้อมใช้ แต่ยังขาด SMTP config และ MSSQL provider ต้องแก้เพิ่มในขั้นตอนถัดไป

ปรับแต่ง docker-compose.yaml สำหรับโปรเจกต์นี้

เปิดไฟล์ docker-compose.yaml แล้วแก้ไข 3 จุดใน block x-airflow-common → environment:

ก) เพิ่ม MSSQL Provider

แทนที่บรรทัด _PIP_ADDITIONAL_REQUIREMENTS (หรือเพิ่มใหม่ถ้าไม่มี):

    _PIP_ADDITIONAL_REQUIREMENTS: >
      apache-airflow-providers-microsoft-mssql>=4.4.0

ข) เพิ่ม SMTP Configuration

เพิ่มใน environment ของ x-airflow-common:

    AIRFLOW__SMTP__SMTP_HOST: ${SMTP_HOST}
    AIRFLOW__SMTP__SMTP_PORT: ${SMTP_PORT}
    AIRFLOW__SMTP__SMTP_STARTTLS: ${SMTP_STARTTLS}
    AIRFLOW__SMTP__SMTP_SSL: ${SMTP_SSL}
    AIRFLOW__SMTP__SMTP_USER: ${SMTP_USER}
    AIRFLOW__SMTP__SMTP_PASSWORD: ${SMTP_PASSWORD}
    AIRFLOW__SMTP__SMTP_MAIL_FROM: ${SMTP_MAIL_FROM}
    AIRFLOW__SMTP__SMTP_TIMEOUT: "60"

ค) เพิ่ม Config File Path และ env_file

เพิ่มใน environment:

    AIRFLOW_CONFIG: '/opt/airflow/config/airflow.cfg'
    AIRFLOW__LOGGING__LOGGING_LEVEL: INFO

เพิ่มระดับเดียวกับ image: (ไม่ใช่ใน environment):

  env_file:
    - .env
✅
ดู docker-compose.yaml ใน repo เป็นตัวอย่าง — ไฟล์นั้นได้ปรับแต่งทุกจุดข้างต้นเรียบร้อยแล้ว

สร้าง Folder Structure ที่จำเป็น

# Linux / macOS / Git Bash
mkdir -p dags/src logs plugins config/secrets
# Windows PowerShell
New-Item -ItemType Directory -Force -Path dags\src, logs, plugins, config\secrets

โครงสร้างที่ต้องมีก่อนรัน:

seamless_data_pipeline/
├── dags/
│   └── src/
├── logs/
├── plugins/
├── config/
│   └── secrets/           ← วาง GCP Service Account JSON ที่นี่
├── .env                   ← สร้างใน Section 4
└── docker-compose.yaml    ← เพิ่งดาวน์โหลดมา

สร้าง .gitignore

# Linux / macOS
cat <<EOF > .gitignore
.env
config/secrets/
logs/
__pycache__/
*.pyc
EOF
# Windows PowerShell
@".env
config/secrets/
logs/
__pycache__/
*.pyc
"@ | Out-File -Encoding utf8 .gitignore
🚨
สำคัญมาก: .env มี credentials และ config/secrets/ มี GCP key — ห้าม commit ขึ้น Git เด็ดขาด

คัดลอก DAG Files เข้าโฟลเดอร์ dags/

วาง DAG files และ src modules ของโปรเจกต์ให้ครบ:

dags/
├── finance_invoice_dag.py
├── staging_student_dag.py
├── student_information_dag.py
├── printer_solution_dag.py
└── src/
    ├── config.py
    ├── dgsi/
    │   ├── finance_invoice/
    │   └── staging_student/
    ├── education/
    │   └── student_information/
    ├── information_tech/
    │   └── printer_solution/
    └── helpers/
        ├── audit.py
        ├── emailer.py
        └── logger.py
✅
เสร็จแล้ว! ข้ามไปที่ Section 4 — ตั้งค่า .env เพื่อตั้งค่า credentials ได้เลย

Path B — มี Repository อยู่แล้ว
git clone <repo-url> seamless_data_pipeline
cd seamless_data_pipeline

โครงสร้างโฟลเดอร์หลักที่ต้องมี:

seamless_data_pipeline/
├── dags/                    ← DAG files + source modules
│   └── src/
│       ├── config.py
│       ├── dgsi/            ← finance_invoice, staging_student

ตั้งค่า .env

ทั้ง Path A และ Path B ทำขั้นตอนนี้ร่วมกัน

คัดลอกจาก .env.example แล้วกรอกค่าจริง:

cp .env.example .env

ค่าที่ต้องแก้ขั้นต่ำ:

# ── Airflow ─────────────────────────────────────────────────
AIRFLOW_UID=50000

# ── Web UI Admin (ใช้ได้เฉพาะก่อนรัน airflow-init ครั้งแรก) ──
_AIRFLOW_WWW_USER_USERNAME=airflow
_AIRFLOW_WWW_USER_PASSWORD=airflow

# ── JWT Secret (optional สำหรับ local dev) ───────────────────
# ถ้าไม่ set จะใช้ค่า default "airflow_jwt_secret"
# แนะนำให้ set สำหรับ shared / production environment
# สร้างด้วย: python -c "import secrets; print(secrets.token_hex(32))"
AIRFLOW__API_AUTH__JWT_SECRET=change_me_to_a_random_secret
AIRFLOW__API_AUTH__JWT_ISSUER=airflow

# ── SMTP ─────────────────────────────────────────────────────
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_STARTTLS=false
SMTP_SSL=true
SMTP_USER=your.email@gmail.com
SMTP_PASSWORD=xxxx xxxx xxxx xxxx
SMTP_MAIL_FROM=Airflow Notification
ALERT_EMAILS=recipient@example.com

# ── Finance Invoice ──────────────────────────────────────────
FIN_INV_TARGET_SCHEMA=dbo
FIN_INV_TARGET_TABLE=finance_invoice_20251110
FIN_INV_KEY_COL=invoiceId
FIN_INV_BATCH_SIZE=1000
⚠️
.env มี credentials ห้าม commit ขึ้น Git!
ตรวจสอบว่า .gitignore มีรายการ .env และ config/secrets/ อยู่แล้ว

AIRFLOW__API_AUTH__JWT_ISSUERairflowoptionalJWT issuer claim — ปกติไม่ต้องเปลี่ยน

รัน Docker Compose

Initialize Airflow (ครั้งแรกเท่านั้น)

docker compose up airflow-init

รอจนเห็น exited with code 0 — ระบบจะ migrate DB และสร้าง admin user อัตโนมัติ

รัน Services ทั้งหมด

docker compose up -d

รอประมาณ 1–2 นาทีจนทุก service แสดงสถานะ healthy

ตรวจสอบ Status

docker compose ps

ทุก service ควรแสดง healthy หรือ running

เข้า Airflow Web UI

เปิด browser ไปที่ http://localhost:8080
username: airflow  ·  password: airflow

(Optional) Flower — Monitor Celery Workers

docker compose --profile flower up -d

เข้าดูที่ http://localhost:5555

⚠️
Windows — AIRFLOW_UID
บน Windows ไม่ต้องเปลี่ยน AIRFLOW_UID จากค่า default 50000 ใน .env

หยุด / Reset

# หยุด containers (เก็บ volumes / data ไว้)
docker compose down

# หยุด + ลบ volumes ทั้งหมด (reset Airflow DB ด้วย)
docker compose down --volumes --remove-orphans
🚨
--volumes จะลบ PostgreSQL metadata ของ Airflow ทั้งหมด (DAG runs, logs, connections, variables) — ใช้เมื่อต้องการ reset ระบบใหม่จาก scratch เท่านั้น

Tips & Troubleshoot

เพิ่ม Python packages

packages ติดตั้งผ่าน _PIP_ADDITIONAL_REQUIREMENTS ใน docker-compose.yaml (ติดตั้งทุกครั้งที่ start) สำหรับ production ควร build custom Docker image แทน

DAG ไม่โชว์ใน UI

docker compose exec airflow-scheduler airflow dags list-import-errors

ดู Log ของ Task

Airflow UI → DAG → Graph View → คลิก Task → แท็บ Log

Trigger DAG ด้วย CLI

docker compose exec airflow-scheduler \
  airflow dags trigger student_information_pipeline

ปัญหา Memory ไม่พอ (Docker Desktop)

Docker Desktop → Settings → Resources → Memory: เพิ่มเป็น ≥ 4 GB แล้ว Apply & Restart

Gmail App Password

Google Account → Security → เปิด 2-Step Verification → App passwords → สร้าง password 16 หลักสำหรับ Airflow (ไม่ใช่ password บัญชีปกติ)

เปลี่ยน JWT Secret ภายหลัง

AIRFLOW__API_AUTH__JWT_SECRET อ่านทุกครั้งที่ container start — เพิ่มหรือเปลี่ยนได้ตลอดเวลา:

Generate random secret

python -c "import secrets; print(secrets.token_hex(32))"

เพิ่มหรือแก้ค่าใน .env

AIRFLOW__API_AUTH__JWT_SECRET=<ค่าที่ได้จากข้อ 1>
AIRFLOW__API_AUTH__JWT_ISSUER=airflow

Restart containers

docker compose down
docker compose up -d
⚠️
JWT token เดิมทั้งหมด invalid ทันที — user ที่ login อยู่ต้อง login ใหม่ แต่ DAG runs และ data ไม่กระทบ

เปลี่ยน Username / Password ภายหลัง

_AIRFLOW_WWW_USER_USERNAME และ _AIRFLOW_WWW_USER_PASSWORD ใช้ได้เฉพาะตอนรัน airflow-init ครั้งแรก — หลังจากนั้นต้องใช้ CLI หรือ UI แทน

กรณีวิธี
ยังไม่เคยรัน airflow-init แก้ .env ตามต้องการ แล้วรัน docker compose up airflow-init ปกติ
รัน airflow-init ไปแล้ว ใช้ CLI หรือ UI ด้านล่าง
ต้องการ reset ทุกอย่าง docker compose down --volumes แล้วแก้ .env และรัน init ใหม่ (ข้อมูล Airflow หาย)

เปลี่ยน Password ผ่าน CLI

docker compose exec airflow-apiserver airflow users reset-password \
  --username airflow \
  --password "<new_password>"
✅
ครอบ password ด้วย "..." เสมอ หาก password มี special characters เช่น * # ! $ — ไม่ครอบอาจถูก shell interpret ผิด

เปลี่ยน Password ผ่าน UI

Admin → Security → List Users → คลิก ✏️ Edit → แก้ช่อง Password → Save

สร้าง User ใหม่ผ่าน CLI

docker compose exec airflow-apiserver airflow users create \
  --username admin \
  --password <password> \
  --role Admin \
  --email admin@example.com \
  --firstname Admin \
  --lastname User
