mssql_student_info port 1433

ฐานข้อมูลระบบพิมพ์เอกสาร PaperCut เก็บ log การใช้เครื่องพิมพ์รายวัน Airflow ดึงข้อมูลมา aggregate เป็นรายเดือน

postgresql_printer_solution port 5432
Airflow Services (Docker Compose)
API Server

ให้บริการ Web UI ที่ :8080 และ REST API สำหรับ trigger DAG, ดู task logs, จัดการ Connections และ Variables ทั้งหมด

localhost:8080
Scheduler

ตรวจ schedule ของทุก DAG อย่างต่อเนื่อง เมื่อถึงเวลาจะสร้าง DAG Run และส่ง task เข้า Redis queue เพื่อให้ Worker รับไปทำงาน

health :8974
DAG Processor

parse ไฟล์ Python ใน ./dags แยก process จาก Scheduler (Airflow 3.x) ช่วยให้ parse error ใน DAG ไม่กระทบ Scheduler หลัก

airflow 3.x
Celery Worker

รับ task จาก Redis queue แล้ว execute จริง — ทุก task ใน pipeline (extract, transform, validate, load, notify) รันที่นี่ สามารถ scale เป็นหลาย worker ได้

CeleryExecutor
Triggerer

จัดการ deferred tasks และ async sensors แบบ event-driven ช่วยลด thread ที่ต้องรอ I/O ให้ Worker ไม่ถูก block

async
Infrastructure
🔴 Redis (Message Broker)

รับ task จาก Scheduler แล้วกระจายไปยัง Celery Worker เป็น in-memory queue ทำให้ส่ง task เร็วและรองรับ Worker หลายตัวพร้อมกัน

port 6379 redis:7.2
🗃️ PostgreSQL (Metadata DB)

เก็บข้อมูลภายในของ Airflow ทั้งหมด — DAG runs, task instances, XCom (ค่าส่งระหว่าง task), Connections, Variables และ User accounts

port 5432 postgres:16
Destinations
🗄️ MSSQL Target (data_op)

ปลายทางสำหรับ 2 pipelines ใช้วิธี MERGE (Upsert) แบบ batch — อัปโหลดข้อมูลเข้า temp table ก่อน แล้ว MERGE เข้า target ทีละ 1,000 rows พร้อม audit log

Finance Invoice Staging Student batch 1,000
☁️ Google BigQuery

ปลายทาง cloud สำหรับ 2 pipelines — Student Info ใช้ MERGE upsert ด้วย key student_id / Printer ใช้ insert-only dedup ด้วย composite key 9 columns

Student Info → Education dataset Printer → PaperCut dataset
Shared Volumes
./dags

DAG Python files และ src/ modules ทั้งหมด — Worker ทุกตัวต้องอ่านไฟล์เดียวกัน การแก้ไขไฟล์มีผลทันทีโดยไม่ต้อง rebuild image

./logs

Task execution logs ทุก run — Airflow UI อ่านจาก volume นี้เพื่อแสดงใน Log tab ของแต่ละ task instance

./config

ไฟล์ airflow.cfg สำหรับ override ค่า config และ secrets/ สำหรับ GCP Service Account JSON key

./plugins

Custom Airflow plugins เช่น Operator หรือ Hook ที่พัฒนาเพิ่มเติม — โหลดอัตโนมัติเมื่อ Airflow เริ่มต้น
