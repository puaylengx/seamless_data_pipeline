ETL Pattern ทั่วไป (ทุก Pipeline)

ทุก DAG ใน project นี้ใช้ pattern เดียวกัน โดยส่งข้อมูลระหว่าง task ผ่าน Parquet file บน shared volume:

mark_start
→
extract_to_file
→
transform
→
validate
→
load
→
notify
Taskหน้าที่Input / Output
mark_start บันทึก timestamp เริ่มต้น → float (Unix timestamp)
extract_to_file ดึงข้อมูลจาก source DB เขียนเป็น Parquet → str (path ไฟล์ raw.parquet)
transform ทำความสะอาด, normalize, type casting raw.parquet → clean.parquet
validate ตรวจสอบ schema, null, ค่าผิดปกติ clean.parquet → dict report
load โหลดข้อมูลเข้า destination (MSSQL/BigQuery) clean.parquet + report → dict stats
notify ส่ง email สรุปผล (inserted/updated/errors) stats dict → email
ℹ️
การใช้ Parquet เป็น intermediate format ช่วยให้ task แต่ละตัว independent — ถ้า task ใดล้มเหลว Airflow จะ retry เฉพาะ task นั้น โดยไม่ต้อง extract ข้อมูลใหม่ทั้งหมด

TaskModuleรายละเอียด

Shared Helpers (dags/src/helpers/)

audit.py — write_audit_line()

บันทึก audit log สำหรับทุก pipeline action (SUMMARY, INSERT, UPDATE) เป็น JSON line file เพื่อ traceability

emailer.py — send_summary_email()

ส่ง email สรุปผลการ run ประกอบด้วย:

Field ใน Emailมาจาก
subjectpipeline name (เช่น "DGSI : Finance Invoice")
statussuccess / no_data / failed
rows_totalจำนวนแถวทั้งหมดที่ดึงได้
inserted / updatedจำนวนแถวที่ insert / update
duration_secเวลาที่ใช้ทั้งหมด (จาก mark_start)
target_tableชื่อ table ปลายทาง
run_dateวันเวลาที่ run (Asia/Bangkok)
updated_samplesตัวอย่าง key ที่ถูก update (สูงสุด 10 รายการ)

อ่าน SMTP config จาก environment variables ใน .env เป็นอันดับแรก

logger.py

Wrapper สำหรับ structured logging — ใช้ logging.getLogger("airflow.task") เพื่อให้ log ปรากฏใน Airflow UI
